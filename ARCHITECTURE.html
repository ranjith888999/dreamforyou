📋 Project Overview

DreamFood is a full-stack food delivery application built with modern web technologies. It provides a seamless experience for users to browse restaurants, explore menus, add items to cart, and complete orders with Google OAuth authentication.

🔐 Google OAuth 2.0 Authentication Flow

1
User Clicks "Sign in with Google"
Frontend renders custom styled Google button
↓
2
Google Popup Authentication
User selects Google account and authenticates
↓
3
ID Token Received
Frontend receives Google ID token from authentication
↓
4
Backend Validation
POST /api/auth/google/token with ID token
Backend validates token with Google using audience parameter
↓
5
User Creation/Update
Backend creates new user or updates existing user
Stores: email, name, google_id, auth_provider
↓
6
JWT Token Generated
Backend generates JWT access token (30 min expiry)
↓
7
User Logged In
Frontend stores JWT token in localStorage
User is redirected to /home

💾 Database Schema

Users Table

Column Type Description
id Integer Primary Key
email String (Unique) User email address
username String (Unique) Unique username
full_name String User's full name
hashed_password String (Nullable) Hashed password (nullable for OAuth users)
google_id String (Unique, Nullable) Google's unique user identifier
auth_provider String Authentication method (email, google, guest)
profile_picture String (Nullable) User profile picture URL
is_active Boolean Account active status

Restaurants Table

Column Type Description
id Integer Primary Key
name String Restaurant name
cuisine_type String Type of cuisine (e.g., Biryani, Pizza)
rating Float Average rating (1-5)
delivery_time Integer Estimated delivery time (minutes)
delivery_charge Float Delivery fee
logo_url String Restaurant logo URL

Menu Items Table

Column Type Description
id Integer Primary Key
restaurant_id Integer (FK) Foreign key to Restaurants
name String Menu item name
price Float Item price
is_vegetarian Boolean Vegetarian flag
spice_level Integer Spice level (1-5)
calories Integer Calorie count

🔧 Critical Fixes & Lessons Learned

Google OAuth Token Validation

Issue: Using incorrect parameter "cid" instead of "audience"

Fix: Changed to verify_oauth2_token(token, request, audience=GOOGLE_CLIENT_ID)

Impact: Enabled successful Google authentication in production

Database Schema Migration

Issue: Database missing google_id column for OAuth users

Fix: Implemented automatic migration on application startup

Impact: Resolved "UndefinedColumn" database errors

Frontend Component Syntax

Issue: Duplicate code in GoogleLoginButton causing syntax errors

Fix: Removed old implementation, kept only custom styled button

Impact: Fixed Docker build failures

Google Button Click Handling

Issue: Custom Google button not triggering OAuth flow

Fix: Implemented proper useRef hook and querySelector approach

Impact: Made Google sign-in button fully functional

Cart Count Display

Issue: Cart items added but count not visible in header

Fix: Added cart badge using getTotalItems() from Zustand store

Impact: Real-time cart count display in header
